Progressive Foundation Engineering (PFE)

Not just dependency layers.

Not just bounded contexts.

A delivery model.

Every Foundation must satisfy all of these before the next Foundation begins:

Specification complete — SPECIFICATION.md defines the domain and invariants.
Public API reviewed — types and contracts are agreed upon before implementation.
Implementation complete — production code exists and matches the specification.
Tests complete — unit tests and invariant tests pass.
Examples complete — at least one example demonstrates correct usage.
Architecture validation — dependency-layer rules are verified.

Only then does that Foundation become "stable," and the team moves to the next one.
